# AI 课程资料知识库 + 学习 Agent 系统

## 项目定位

本项目面向大学课程复习场景，目标是把课件、PDF、实验报告、课堂笔记等资料构建成可检索、可追问、可出题的课程知识库。系统不是普通聊天机器人，而是一个围绕学习任务设计的 AI 应用产品：用户上传资料后，可以进行资料问答、知识点总结、自动出题、复习计划生成和错题回顾。

## 技术栈

后端使用 Python + FastAPI 提供 API 服务。知识库层负责文档解析、文本清洗、chunk 切分、索引构建、混合检索和重排。前端使用浏览器页面作为 demo 控制台，展示问答、引用来源、工具调用轨迹、检索分数和响应指标。系统保留 OpenAI-compatible LLM 适配层，可以接入 OpenAI、DeepSeek、Qwen、混元等模型服务。

## RAG 主链路

RAG 链路包括资料上传、文本解析、chunk 切分、向量化或 TF-IDF 表示、Top-K 召回、Rerank 二次排序、上下文拼接、答案生成和来源引用。系统要求回答优先基于资料，资料不足时明确说明缺口，避免把低相关片段硬解释成答案。

## Agent 架构

系统采用 Router + Tool 的轻量 Agent 架构。Router 先识别用户意图，例如资料问答、总结、出题、概念解释、复习计划、错题回顾、项目包装和评估说明。随后 Agent 选择对应工具链：hybrid_retrieval、rerank、summarize_tool、generate_quiz、explain_concept、make_review_plan、mistake_review 等。最后由 answer_synthesizer 组织答案并更新 session memory。

## 答案质量策略

答案生成需要先判断证据强度。高相关资料命中时，回答应给出结论、关键依据和来源。低相关资料命中时，系统应提示“当前知识库直接证据不足”，并引导用户补充章节、上传资料或改问更具体概念。对于 RAG、Agent、Embedding、Rerank、OpenAI、Prompt 工程等项目概念，系统可使用内置知识卡片兜底，但要说明这不是上传资料的直接证据。

## 评估指标

检索层可以评估 Top-K 命中率、MRR、召回片段相关性和 Rerank 后排序变化。生成层可以评估答案忠实度、引用覆盖率、幻觉率和人工满意度。Agent 层可以评估意图识别准确率、工具选择准确率、平均工具调用步数和失败恢复能力。工程层可以评估响应延迟、知识库加载时间、上传解析成功率和 API 成功率。
