# AI 原生工程核心概念

## RAG

RAG 是 Retrieval-Augmented Generation，中文通常叫检索增强生成。它先从知识库中检索与问题相关的资料片段，再把这些片段作为上下文交给大模型生成答案。RAG 适合企业知识库、课程资料问答、客服问答和私有文档助手，因为它能让回答基于外部资料，并通过引用来源降低幻觉风险。

## Agent

Agent 是能围绕目标自主选择动作并调用工具的应用架构。一个工程上可落地的 Agent 不一定需要复杂多智能体，可以从 Router + Tool 开始：先判断用户意图，再选择检索、总结、出题、复习计划、错题回顾等工具。Agent 的关键能力包括 Tool Use、Planning、Memory、状态追踪和失败恢复。

## Embedding

Embedding 是把文本、图片或其他对象映射为向量表示的方法。文本 Embedding 可以表达语义相似度，使“事务隔离级别”和“数据库并发控制”这类相关问题在向量空间中更接近。知识库系统通常把文档 chunk 和用户问题都转成向量，然后用相似度检索相关资料。

## 向量数据库

向量数据库用于存储和检索 embedding 向量，常见选择包括 FAISS、Chroma、Milvus、Elasticsearch 向量检索等。向量数据库解决的是相似语义召回问题，但它不等于完整 RAG 系统；完整 RAG 还需要切分策略、元数据管理、Rerank、Prompt 组织、答案生成和评估。

## Rerank

Rerank 是对初次召回结果进行二次排序。向量检索擅长召回语义相近片段，但排序不一定最准确，尤其是中文课程资料中存在专业名词、公式、章节标题和上下文依赖。Rerank 可以融合向量相似度、关键词覆盖、章节标题命中、来源可信度和结构化规则，把最能回答问题的 chunk 放到前面。

## Prompt 工程

Prompt 工程不是简单写一句“请回答问题”，而是设计任务指令、上下文格式、输出结构、拒答边界和引用规范。在学习 Agent 中，问答 Prompt 应强调基于资料回答，总结 Prompt 应强调提炼复习重点，出题 Prompt 应生成题干、选项、答案和解析，错题回顾 Prompt 应定位薄弱知识点并推荐复习资料。

## OpenAI

OpenAI 是提供 GPT 系列大模型、Embedding、语音、多模态等 AI API 的公司和平台。在本项目里，OpenAI 可以作为可插拔模型供应商之一，用于 LLM 生成或语义向量化。项目不应与单一模型供应商强绑定，工程上应保留 OpenAI-compatible API 适配层，以便切换 DeepSeek、Qwen、混元等服务。

## Tool Use、Planning 与 Memory

Tool Use 指 Agent 能调用外部工具，例如检索知识库、生成练习题、制定复习计划。Planning 指 Agent 能根据目标拆分步骤，例如先识别意图，再检索资料，再生成答案。Memory 指系统保留一定范围内的对话上下文，使用户追问“这个再解释一下”时可以关联上文。学习类产品通常适合短期 session memory，而不是默认永久记忆。
